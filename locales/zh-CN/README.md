# 三令四卡法 (T3C4) — 中文语言包

> "开工/收工/刷新"三令 + "一张一事"四段卡片 = 多 Agent 项目协同标准操作

## 中文触发词

| 你说 | Agent 做什么 |
|------|-------------|
| **"开工"** | 读 CURRENT.md → 读最近 3 张卡片 → 报告当前状态 → 等你下指令 |
| **"收工"** / "完事了" | 写新卡片 → 更新 CURRENT.md → 总结做了什么 |
| **"刷新上下文"** / "refresh" | 检查 CURRENT.md 时间戳 → 重新读变更 → 读最新卡片 → 报告 |
| **"看看进度"** / "status" | 读 CURRENT.md → 报告实验状态、阻塞项、下一步 |
| **"记一下"** | Agent 把刚才的发现写成一张卡片 |

## 新项目启动 6 步

1. 复制 `_templates/project-workflow/` → 你的项目目录
2. 打开 AGENTS.md 和 CURRENT.md，把 `[项目名称]` 替换为你的项目名
3. 填写 CURRENT.md：当前目标 + 任务队列 + 已知风险
4. 写第一批卡片（项目目标、数据概览、技术要求）
5. 说 **"开工"**
6. 里程碑时编译到个人知识库

## 核心思想

```
所有 Agent 共享同一份文件，就是共享同一份记忆。
```

文件系统即共享总线。开工读，收工写。不需要服务器，不需要 API，不需要加密密钥。

## 卡片纪律

- **一张一事**：一张卡片只记录一个发现/决策/风险
- **先事实后判断**：先写看到了什么，再写这意味着什么
- **说"记一下"就写**：Agent 发现坑 → 自动写卡。你想记东西 → 说"记一下"
- **收工批量写入**：干完活说"收工"，Agent 写所有该写的卡片

## 多 Agent 协调

| Agent | 最适合的任务 |
|-------|------------|
| **本地 Codex/Claude Code** | 写代码、跑实验、数据分析、调参数 |
| **远程 cc-connect** | 查看进度、读日志、写卡片、轻量修改 |
