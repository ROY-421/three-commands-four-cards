# Cards Index

This layer stores minimal-granularity information units.

## Card Principles

- One card, one fact
- Facts first, judgments second
- Keep cards short
- Include source when traceability matters

## Card Types

- `decision` — Decision record
- `task` — Task tracking
- `risk` — Risk identification
- `meeting` — Meeting / discussion notes
- `finding` — Discovery (data format, experiment result, technical insight)
- `reference` — Reference information
